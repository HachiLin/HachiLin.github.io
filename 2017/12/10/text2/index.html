hexo+github搭建个人博客

个人笔记
  • hexo


##hexo+github搭建个人博客

一.基础博客搭建流程

1.1 安装node和git

1.安装node.js（Node.js中文官网）。自己根据情况安装合适的版本。

2.在本机安装git（Git官网)。根据情况安装合适的版本。安装正确的话，在桌面或文件夹空白处鼠标右键菜单会新增“Git GUI Here”和“Git Bash Here”两个选项。

3.安装npm，淘宝cnpm官网。由于npm国内下载速度经常抽风，所以建议安装淘宝的这个镜像；使用方法就是在命令中把npm换成cnpm即可。
安装方法：在cmd工具中输入如下命令

npm install -g cnpm –registry=https://registry.npm.taobao.org

4.注册github（详细的使用及管理可以自主去学习，这里就不再陈述）。

5.当这三者都安装完后可以通过cmd（我使用的是windows）查看自己是否安装成功。在cmd中执行以下步骤：

node -v
npm -v
git –version



cmd 运行结果

1.2 安装Hexo

1.新建一个文件夹如blog。

2.进入blog文件夹，右键点击 Git Bash Here。

3.安装Hexo，输入代码

$ cnpm install hexo-cli -g

4.初始化blog文件夹

$ hexo init

5.安装依赖和插件

$ cnpm install

$ cnpm install hexo-server –save #搭建本地服务器所需插件

$ cnpm install hexo-deployer-git –save #使用git方式进行部署博客所需插件


1.3 在本地生成博客静态页面并预览

1.在本地生成静态网页

$ hexo generate

此时打开blog文件夹会发现一个用来存放静态文件的public的文件，该命令可简写为$ hexo g

2.启动本地服务器

$ hexo server

这条指令运行完成后可在本地启动服务器并预览博客，默认网址为http://localhost:4000/。

如果以上步骤都不出意外的话，在浏览器中打开该网址，就会看到一个Hexo博客初始化的页面。改命令可简写为$ hexo s


1.4 在github上创建并设置远程库

1.创建博客仓库
假如你注册的用户名是Tom，那么新建一个名为Tom.github.io的仓库，注意仓库的用户名一定要是用户名+github.io的形式！创建之后，点击setting设置远程库：
找到GitHub Pages点击change theme选个主题select theme设置模板
等待github pages创建完成，然后输入你创建的地址测试github pager是否创建成功。此时页面是空的没有任何东西。

2.编辑博客配置文件: _config.yml
在hexo根目录（也就是blog文件夹）下找到_config.yml文件，把其中的deploy参数（没有的话就按如下格式新建，注意冒号后面一定要有一个空格），修改为：

deploy:
type: git
repo: git@github.com:Tom/Tom.github.io.git
branch: master

3.重新部署
在博客根目录下打开Git Bash，依次执行如下Hexo命令：

$ hexo clean #会清除缓存文件db.json及之前生成的静态文件夹public；

$ hexo g #会重新生成静态文件夹public；

$ hexo deploy #因为之前已经安装了插件并且在博客配置文件中也配置好了，所以这个命令会在博客根目录下生成一个.deploy_git的文件夹，并 把本地生成的静态文件部署到Tom.github.io这个仓库中的master分支上，该命令可简写为hexo d；

4.在浏览器中访问博客
在浏览器中输入Tom.github.io（可能你已经发现了，这个就是之前新建仓库的名字，同时也是你博客的域名），没问题的话，你应该可以再次看到那个熟悉又亲切的博客页面了。
